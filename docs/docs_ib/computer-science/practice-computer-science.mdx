<PracticeProblem
  question="What is the time complexity of binary search on a sorted array of n elements?"
  options={['O(log n)', 'O(n)', 'O(n log n)', 'O(1)']}
  correctAnswer={0}
  explanation="Binary search works by repeatedly dividing the sorted array in half. With each comparison, it eliminates half of the remaining elements. Starting with n elements, after k comparisons, n/2^k elements remain. The search terminates when n/2^k = 1, so k = log2(n). This gives O(log n) time complexity. For example, searching 1,000,000 elements requires at most about 20 comparisons. This is significantly more efficient than linear search's O(n) but requires the array to be pre-sorted."
  difficulty="medium"
