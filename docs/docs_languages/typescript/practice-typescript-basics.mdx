### Generics

### Enums and Union Types

<PracticeProblem
  question={`What is a key advantage of using \`type Status = 'active' | 'inactive' | 'pending'\` over \`enum Status { Active, Inactive, Pending }\`?`}
  options={[
    'Union types provide exhaustiveness checking in switch statements without emitting runtime JavaScript code',
    'Union types are faster at runtime',
    'Union types can be iterated with `for...of`',
    'Union types support reverse mapping from value to name',
  ]}
  correctAnswer={0}
  explanation="Union literal types exist only at the type level and are erased at runtime, producing zero JavaScript overhead. They still enable exhaustive checking in switch statements, especially with a `never` default case. Enums generate runtime JavaScript code (an IIFE and lookup object), which adds to bundle size. The TypeScript team recommends union types for most cases, reserving enums for when you need runtime enumeration values or reverse mapping."
  difficulty="medium"
/>
