### Optionals

<PracticeProblem
  question="What is the value of `let result: Int? = nil ?? 42`?"
  options={['nil', '42', 'Compile error', 'Runtime crash']}
  correctAnswer={1}
  explanation="The nil coalescing operator `??` unwraps the left side. Since it is `nil`, the default value `42` is used. The result type is `Int` (not optional)."
  difficulty="medium"
/>

<PracticeProblem
  question='Which statement correctly safely unwraps an optional `let name: String? = "Alice"`?'
  options={['let n = name!', 'if let n = name { }', 'let n: String = name', 'name.bind { n in }']}
  correctAnswer={1}
  explanation="`if let` (optional binding) safely unwraps without crashing. `!` force-unwraps and crashes if nil. Direct assignment from optional to non-optional is a compile error."
  difficulty="easy"
/>

### OOP and Error Handling

<PracticeProblem
  question="What is the output of the following code? `struct Point { var x: Int };

var a = Point(x: 1); var b = a; b.x = 2; print(a.x)`"
  options={['1', '2', 'nil', 'Compile error']}
  correctAnswer={0}
  explanation="Structs are value types. Assigning `a` to `b` creates a copy. Modifying `b.x` does not affect `a`. This is different from classes, which share the same reference."
