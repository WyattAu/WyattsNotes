### Types and Variables

<Flashcard
  front="What is the difference between `let` and `var` in Swift?"
  back="`let` declares a constant (immutable), `var` declares a variable (mutable). Prefer `let` unless the value must change."
/>

<Flashcard
  front="Does Swift infer variable types? Give an example."
  back='Yes. `let name = "Alice"` infers `String`. `let count = 42` infers `Int`. You can also be explicit: `let score: Double = 95.0`.'
/>

<Flashcard
  front="What are Swift's integer types and how is `Int` sized?"
  back="`Int8`, `Int16`, `Int32`, `Int64`, and `UInt` variants. `Int` maps to the platform's native word size: `Int64` on 64-bit, `Int32` on 32-bit."
/>

<Flashcard
  front="What is a tuple in Swift and when would you use one?"
  back="A tuple groups multiple values of different types: `let result = (success: true, code: 200)`. Useful for returning multiple values from a function without defining a struct."
/>

### Optionals

<Flashcard
  front="What is an optional in Swift and what values can it hold?"
  back='An optional (`T?`) can hold either a value of type `T` or `nil`. Example: `var name: String? = "Alice"`. It is Swift&apos;s mechanism for safely handling the absence of a value.'
/>

<Flashcard
  front="What are four ways to unwrap an optional?"
  back="1) Forced unwrap with `!` (crashes if nil), 2) Optional binding `if let`, 3) Guard let `guard let`, 4) Nil coalescing `??`. Always prefer the safe methods over `!`."
/>

<Flashcard
  front="What does the nil coalescing operator `??` do?"
  back='`a ?? b` returns `a` if it is non-nil; otherwise returns `b`. Example: `let name = username ?? "Guest"`. `b` must match the type of `a`&apos;s unwrapped value.'
/>

<Flashcard
  front="What is optional chaining and how does it work?"
  back="Optional chaining (`?.`) accesses properties/methods on an optional. If the optional is nil, the entire expression returns nil instead of crashing. Example: `let street = person.address?.street`."
/>

### Collections

<Flashcard
  front="What are the three main collection types in Swift?"
  back="Array (ordered, indexable), Dictionary (key-value pairs, unordered), and Set (unordered, unique elements). All are value types with copy-on-write optimisation."
/>

<Flashcard
  front="How does `map`, `filter`, and `reduce` work on arrays?"
  back="`map` transforms each element. `filter` keeps elements matching a condition. `reduce` combines all elements into a single value. Example: `[1,2,3].map {$0*2}` yields `[2,4,6]`. `[1,2,3].filter {$0>1}` yields `[2,3]`. `[1,2,3].reduce(0,+)` yields `6`."
/>

<Flashcard
  front="What is the difference between `compactMap` and `flatMap`?"
  back="`compactMap` transforms and removes nil results from optional-returning closures. `flatMap` flattens nested arrays or removes nils (same as `compactMap` on optionals)."
/>

### Functions and Closures

<Flashcard
  front="What is a closure in Swift?"
  back='A self-contained block of code that can capture values from its surrounding context. Closures are anonymous functions. Example: `let greet = { (name: String) in "Hello, \(name)" }`.'
/>

<Flashcard
  front="What are shorthand argument names in closures?"
  back="Swift provides `$0`, `$1`, `$2` etc. as shorthand for closure parameters. Example: `names.sorted(by: { $0.count < $1.count })` instead of writing `(a, b) in a.count < b.count`."
/>

<Flashcard
  front="What is an escaping closure and when is `@escaping` required?"
  back="A closure stored or executed after the function returns. Requires `@escaping` annotation. Common with async callbacks, completion handlers, and stored property closures. Non-escaping closures execute before the function returns."
/>

<Flashcard
  front="What does `@autoclosure` do?"
  back="`@autoclosure` wraps an expression in a closure, deferring evaluation until the closure is called. Used by `assert()` and `??`. Example: `assert(debugMode)` does not evaluate `debugMode` until inside assert."
/>

### OOP and Protocols

<Flashcard
  front="What is the key difference between a struct and a class?"
  back="Structs are value types (copied on assignment), have no inheritance, and get automatic memberwise init. Classes are reference types (shared), support inheritance, and require manual init."
/>

<Flashcard
  front="What is a protocol in Swift and how does it differ from a class?"
  back="A protocol defines a blueprint of requirements (methods, properties) that conforming types must implement. Unlike a class, it cannot store properties -- only define them. Multiple protocols can be adopted."
/>

<Flashcard
  front="What is a property wrapper in Swift?"
  back="A property wrapper encapsulates get/set logic in a reusable wrapper. Common SwiftUI examples: `@State`, `@Binding`, `@ObservedObject`, `@EnvironmentObject`, `@Published`."
/>

<Flashcard
  front="How does Automatic Reference Counting (ARC) work?"
  back="ARC tracks the number of strong references to each class instance. When the count reaches zero, the instance is deallocated. Use `weak` or `unowned` to break strong reference cycles."
/>
