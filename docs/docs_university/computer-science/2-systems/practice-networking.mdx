## Network Models and Physical/Data Link Layers

<PracticeProblem question="In the OSI model, which layer is responsible for reliable end-to-end data transfer between processes?" options={[ 'Transport layer (Layer 4)', 'Network layer (Layer 3)', 'Session layer (Layer 5)', 'Data link layer (Layer 2)', ]} correctAnswer={0} explanation="The transport layer (Layer 4) provides end-to-end communication services including reliable delivery (TCP), flow control, congestion control, and multiplexing. The network layer (Layer 3) handles routing and logical addressing (IP). The session layer manages dialogues. The data link layer handles framing and error detection on a single hop." difficulty="easy" />

<PracticeProblem question="Shannon's theorem for a noisy channel gives the maximum data rate as $C = B \\log_2(1 + \\text{SNR})$. If SNR = 30 dB, what is the numerical SNR value used in the formula?" options={[ '$\\text{SNR} = 10^{30/10} = 1000$', '$\\text{SNR} = 30$', '$\\text{SNR} = 10^{30} $', '$\\text{SNR} = \\log_2(30)$', ]} correctAnswer={0} explanation="SNR in decibels is defined as $\\text{SNR}_{\\text{dB}} = 10 \\log_{10}(\\text{SNR}_{\\text{power ratio}})$. To convert back: $\\text{SNR} = 10^{\\text{SNR}_{\\text{dB}}/10}$. For 30 dB: $\\text{SNR} = 10^3 = 1000$. If bandwidth is $B$ Hz, then $C = B \\log_2(1001) \\approx 10B$. A 30 dB SNR means the signal power is 1000 times the noise power." difficulty="medium" />

<PracticeProblem question="CRC (Cyclic Redundancy Check) uses polynomial division to detect errors. A CRC with generator polynomial of degree $r$ can detect all error bursts of length up to:" options={['$r$ bits', '$r/2$ bits', '$r + 1$ bits', '$2r$ bits']} correctAnswer={0} explanation="CRC appends $r$ zero bits to the message and divides by a generator polynomial $G(x)$ of degree $r$. The remainder (also $r$ bits) is appended as the CRC. This construction guarantees detection of all single-bit errors, all double-bit errors, all odd-number errors, and any burst error of length $\\leq r$ bits. For bursts longer than $r$, the detection probability is $1 - 2^{-r}$." difficulty="medium" />

<PracticeProblem question="In CSMA/CD (used in Ethernet), what happens when a collision is detected?" options={[ 'A jam signal is sent, then both stations wait a random backoff time using binary exponential backoff before retrying', 'The station with the lower MAC address wins priority', 'Both stations immediately retransmit', 'The collision is resolved by the switch', ]} correctAnswer={0} explanation="On collision detection, both stations send a jam signal to ensure all nodes detect the collision, then stop transmitting. Each station waits a random backoff time: $\\text{backoff} = \\text{rand}(0, 2^k - 1) \\times \\text{slot time}$ where $k = \\min(\\text{retries}, 10)$. The exponential backoff reduces collision probability on repeated attempts." difficulty="easy" />

<PracticeProblem question="What is the key difference between CSMA/CD (Ethernet) and CSMA/CA (Wi-Fi)?" options={[ 'CSMA/CA avoids collisions using RTS/CTS handshake because Wi-Fi cannot detect collisions during transmission', 'CSMA/CD avoids collisions; CSMA/CA only detects them', 'CSMA/CD is used for wireless networks; CSMA/CA for wired networks', 'Both are identical in operation', ]} correctAnswer={0} explanation="Ethernet (wired) uses CSMA/CD: stations can detect collisions while transmitting by comparing signal voltage on the wire. Wi-Fi (wireless) cannot reliably detect collisions during transmission due to the hidden terminal problem. CSMA/CA avoids collisions by reserving the channel with RTS/CTS (Request to Send / Clear to Send) handshake before data transmission." difficulty="medium" />

## Network Layer, Transport Layer, and Application Layer

<PracticeProblem question="Given the network 192.168.1.0/24, what is the subnet mask and the number of usable hosts per subnet if split into /26 subnets?" options={[ 'Subnet mask 255.255.255.192; 62 usable hosts per subnet', 'Subnet mask 255.255.255.128; 126 usable hosts per subnet', 'Subnet mask 255.255.255.224; 30 usable hosts per subnet', 'Subnet mask 255.255.255.0; 254 usable hosts per subnet', ]} correctAnswer={0} explanation="A /26 prefix means 26 bits for network, 6 bits for host. Subnet mask: 11111111.11111111.11111111.11000000 = 255.255.255.192. Each subnet has $2^6 = 64$ addresses, minus network address and broadcast address = 62 usable hosts. From /24, this creates 4 subnets: 192.168.1.0/26, 192.168.1.64/26, 192.168.1.128/26, 192.168.1.192/26." difficulty="medium" />

<PracticeProblem question="In TCP congestion control, what happens when three duplicate ACKs are received (fast retransmit)?" options={[ 'cwnd is halved (set to ssthresh) and the missing segment is retransmitted immediately', 'cwnd is reset to 1 MSS and slow start begins', 'The connection is terminated', 'No action is taken; the receiver must request retransmission', ]} correctAnswer={0} explanation="Three duplicate ACKs indicate that one segment was lost but subsequent segments arrived. This triggers fast retransmit: retransmit the missing segment immediately (without waiting for timeout), set ssthresh = cwnd/2, and set cwnd = ssthresh (fast recovery). This avoids the costly timeout mechanism that would reset cwnd to 1 MSS." difficulty="hard" />

<PracticeProblem question="In DNS resolution, which record type maps a domain name to an alias (another domain name)?" options={['CNAME (Canonical Name)', 'A (IPv4 address)', 'MX (Mail Exchange)', 'NS (Name Server)']} correctAnswer={0} explanation="CNAME maps a domain to another domain (aliasing). For example, www.example.com might be a CNAME for example.com. A records map to IPv4 addresses directly. MX records specify mail servers. NS records specify authoritative name servers for a zone. The DNS resolver follows CNAME chains until reaching an A or AAAA record." difficulty="easy" />

<PracticeProblem question="What is the primary advantage of HTTP/2 over HTTP/1.1?" options={[ 'Multiplexing: multiple requests and responses over a single TCP connection without head-of-line blocking at the application layer', 'UDP transport instead of TCP', 'No encryption required', 'Simpler header format without compression', ]} correctAnswer={0} explanation="HTTP/1.1 allows pipelining but suffers from head-of-line blocking: one slow response blocks all subsequent responses on the same connection. HTTP/2 multiplexes requests and responses on a single TCP connection using streams and frames, with HPACK header compression. However, HTTP/2 still has HOL blocking at the TCP layer (solved by HTTP/3 with QUIC/UDP)." difficulty="medium" />

<PracticeProblem question="TCP uses a sliding window for flow control. What is Silly Window Syndrome and how is it avoided on the sender side?" options={[ "Small segments are sent due to tiny advertised windows; Nagle's algorithm buffers small writes and sends them together", 'Large segments overflow the receiver buffer; the window size must be doubled', 'The window closes permanently; the connection must be reset', 'Segments arrive out of order; selective acknowledgments fix this', ]} correctAnswer={0} explanation="Silly Window Syndrome occurs when the receiver advertises tiny windows (e.g., 1 byte) causing the sender to transmit tiny segments inefficiently. Nagle’s algorithm (sender side) buffers small writes until an ACK for the previous data arrives or enough data accumulates for a full segment. Clark's solution (receiver side) avoids advertising small windows." difficulty="hard" />
