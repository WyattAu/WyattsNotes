### Synchronization and File Systems

<PracticeProblem
  question="What is the primary purpose of a mutex (a binary semaphore initialized to 1)?"
  options={[
    'Ensure mutual exclusion: only one process executes in a critical section at a time',
    'Allow multiple processes to access a shared resource simultaneously',
    'Prevent deadlock by eliminating the circular wait condition',
    'Implement priority inheritance for real-time scheduling',
  ]}
  correctAnswer={0}
  explanation="A mutex provides mutual exclusion for a critical section. A process must acquire the lock (wait on the semaphore) before entering and release it (signal) upon leaving. Since the semaphore is initialized to 1, only one process can decrement it to 0 and enter; others block until it is released back to 1. Mutexes differ from general semaphores in that they are designed specifically for mutual exclusion -- the releasing process must be the same as the acquiring process (ownership), and they often support priority inheritance to prevent priority inversion."

/>
